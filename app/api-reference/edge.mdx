title: Edge Runtime
description: API reference for the Edge Runtime used in Next.js, including supported Web APIs and restrictions.
---

Next.js has two server runtimes:

- The **Node.js Runtime** (default): Access to all Node.js APIs, used for rendering your application.
- The **Edge Runtime**: A more limited set of Web APIs, used in Middleware and Edge routes.

## Caveats

- The Edge Runtime does not support all Node.js APIs. Some packages may not work as expected.
- The Edge Runtime does not support Incremental Static Regeneration (ISR).
- Both runtimes can support [streaming](/app/api-reference/file-conventions/loading) depending on your deployment adapter.

## Unsupported APIs

- Native Node.js APIs are **not supported** (e.g., filesystem access).
- `node_modules` can be used if they implement ES Modules and don't use native Node.js APIs.
- Calling `require` directly is **not allowed**. Use ES Modules.

The following JavaScript features are disabled:

| API | Description |
| --- | --- |
| [`eval`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/eval) | Evaluates JavaScript code as a string |
| [`new Function(evalString)`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Function) | Creates a function from a string |
| [`WebAssembly.compile`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/compile) | Compiles a WebAssembly module from a buffer |
| [`WebAssembly.instantiate`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/instantiate) | Compiles and instantiates from a buffer |

If your code contains unreachable dynamic code evaluation statements, you can allow them in your Proxy configuration:

```javascript filename="proxy.ts"
export const config = {
  unstable_allowDynamic: [
    '/lib/utilities.js',
    '**/node_modules/function-bind/**',
  ],
}
```

<Warning>If these statements are executed on the Edge, they will throw a runtime error.</Warning>
