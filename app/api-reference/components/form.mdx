The `<Form>` component extends the HTML `<form>` element to provide **prefetching** of loading UI, **client-side navigation** on submission, and **progressive enhancement**.

It's useful for forms that update URL search params, as it reduces the boilerplate needed for these patterns.

```tsx filename="/app/ui/search.tsx"
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      {/* On submission, the input value will be appended to
          the URL, e.g. /search?query=abc */}
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

## Behavior

The behavior of `<Form>` depends on whether `action` is a string or a function:

- **`action` is a string**: The form behaves like a native HTML form using a `GET` method. Form data is encoded into the URL as search params. Next.js also prefetches the path when the form becomes visible, and performs client-side navigation instead of a full page reload on submission.
- **`action` is a function (Server Action)**: The form behaves like a React form, executing the action on submission.

## Props

### String `action` props

<ParamField path="action" type="string" required>
  The URL or path to navigate to when the form is submitted. An empty string `""` navigates to the same route with updated search params.
</ParamField>

<ParamField path="replace" type="boolean">
  Replaces the current history state instead of pushing a new entry. Defaults to `false`.
</ParamField>

<ParamField path="scroll" type="boolean">
  Controls scroll behavior on navigation. Defaults to `true` (scrolls to top of new route).
</ParamField>

<ParamField path="prefetch" type="boolean">
  Controls whether the path is prefetched when the form becomes visible in the viewport. Defaults to `true`.
</ParamField>

### Function `action` props

<ParamField path="action" type="function" required>
  The Server Action to call when the form is submitted. See the [React docs](https://react.dev/reference/react-dom/components/form#props) for more.
</ParamField>

<Note>When `action` is a function, the `replace` and `scroll` props are ignored.</Note>

## Caveats

- **`formAction`**: Can be used in `<button>` or `<input type="submit">` to override the `action` prop. Does not support prefetching.
- **`key` prop**: Passing a `key` to a string `action` is not supported. Use a function `action` instead.
- **`onSubmit`**: Can handle form submission logic, but calling `event.preventDefault()` overrides `<Form>` navigation behavior.
- **`method`, `encType`, `target`**: Not supported as they override `<Form>` behavior. Use an HTML `<form>` instead.
- **`<input type="file">`**: Submits the filename rather than the file object when `action` is a string.

## Examples

### Search form

```tsx filename="/app/page.tsx"
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

On the results page, access the query using the `searchParams` prop:

```tsx filename="/app/search/page.tsx"
import { getSearchResults } from '@/lib/search'

export default async function SearchPage({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const results = await getSearchResults((await searchParams).query)
  return <div>...</div>
}
```

When the form becomes visible, shared UI on `/search` is prefetched. On submission, the form immediately navigates and shows `loading.js` while results load.

To show instant feedback using `useFormStatus`:

```tsx filename="/app/ui/search-button.tsx"
'use client'
import { useFormStatus } from 'react-dom'

export default function SearchButton() {
  const status = useFormStatus()
  return (
    <button type="submit">{status.pending ? 'Searching...' : 'Search'}</button>
  )
}
```

### Mutations with Server Actions

```tsx filename="/app/posts/create/page.tsx"
import Form from 'next/form'
import { createPost } from '@/posts/actions'

export default function Page() {
  return (
    <Form action={createPost}>
      <input name="title" />
      <button type="submit">Create Post</button>
    </Form>
  )
}
```
