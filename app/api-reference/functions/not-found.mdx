title: notFound
description: API reference for the notFound function, which renders the not-found UI for a route segment.
---

The `notFound` function renders the [`not-found.js` file](/app/api-reference/file-conventions/not-found) within a route segment and injects a `<meta name="robots" content="noindex" />` tag.

## Usage

```ts
import { notFound } from 'next/navigation'
```

Invoking `notFound()` throws a `NEXT_HTTP_ERROR_FALLBACK;404` error and terminates rendering of the route segment.

```tsx app/user/[id]/page.tsx
import { notFound } from 'next/navigation'

async function fetchUser(id: string) {
  const res = await fetch(`https://api.example.com/users/${id}`)
  if (!res.ok) return undefined
  return res.json()
}

export default async function Profile({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  const user = await fetchUser(id)

  if (!user) {
    notFound()
  }

  return <div>{user.name}</div>
}
```

<Note>
`notFound()` does not require `return notFound()` because it uses the TypeScript `never` type.
</Note>

## Good to know

- `notFound()` can be called in Server Components, Client Components, Route Handlers, and Server Actions.
- Create a [`not-found.js`](/app/api-reference/file-conventions/not-found) file to customize the 404 UI shown when `notFound()` is called.
- If no `not-found.js` is present in the segment, Next.js will propagate to the nearest parent `not-found.js`.
