title: cookies
description: API reference for the cookies function, which reads and writes HTTP cookies in Server Components, Server Functions, and Route Handlers.

`cookies` is an **async** function that reads HTTP incoming request cookies in [Server Components](/app/getting-started/server-and-client-components), and reads/writes outgoing request cookies in [Server Functions](/app/getting-started/mutating-data) or [Route Handlers](/app/api-reference/file-conventions/route).

```tsx filename="app/page.tsx"
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

## Methods

| Method | Return type | Description |
| --- | --- | --- |
| `get('name')` | Object | Accepts a cookie name and returns an object with the name and value. |
| `getAll()` | Array of objects | Returns a list of all cookies with a matching name, or all cookies if no name is given. |
| `has('name')` | Boolean | Returns a boolean based on whether the cookie exists. |
| `set(name, value, options)` | — | Sets the outgoing request cookie. |
| `delete(name)` | — | Deletes the cookie. |
| `toString()` | String | Returns a string representation of the cookies. |

## Options

When setting a cookie, the following `options` properties are supported:

| Option | Type | Description |
| --- | --- | --- |
| `name` | String | Specifies the name of the cookie. |
| `value` | String | Specifies the value to store. |
| `expires` | Date | Defines when the cookie expires. |
| `maxAge` | Number | Cookie lifespan in seconds. |
| `domain` | String | The domain where the cookie is available. |
| `path` | String (default: `'/'`) | Limits the cookie scope to a specific path. |
| `secure` | Boolean | Only send over HTTPS connections. |
| `httpOnly` | Boolean | Restricts to HTTP requests, preventing client-side access. |
| `sameSite` | Boolean \| `'lax'` \| `'strict'` \| `'none'` | Controls cross-site request behavior. |
| `priority` | `"low"` \| `"medium"` \| `"high"` | Cookie priority. |
| `partitioned` | Boolean | Whether the cookie is [partitioned](https://github.com/privacycg/CHIPS). |

## Good to know

- `cookies` is an **asynchronous** function returning a promise. Use `async/await` or React's `use` function.
- `cookies` is a request-time API. Using it opts the route into dynamic rendering.
- The `.delete` method can only be called in a Server Function or Route Handler, and only for the same domain from which `.set` was called.
- HTTP does not allow setting cookies after streaming starts. Use `.set` in a Server Function or Route Handler.

## Examples

### Getting a cookie

```tsx filename="app/page.tsx"
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

### Getting all cookies

```tsx filename="app/page.tsx"
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  return cookieStore.getAll().map((cookie) => (
    <div key={cookie.name}>
      <p>Name: {cookie.name}</p>
      <p>Value: {cookie.value}</p>
    </div>
  ))
}
```

### Setting a cookie

```ts filename="app/actions.ts"
'use server'

import { cookies } from 'next/headers'

export async function create(data: FormData) {
  const cookieStore = await cookies()
  cookieStore.set('name', 'lee')
  // or
  cookieStore.set('name', 'lee', { secure: true })
  // or
  cookieStore.set({
    name: 'name',
    value: 'lee',
    httpOnly: true,
    path: '/',
  })
}
```

### Deleting a cookie

```ts filename="app/actions.ts"
'use server'

import { cookies } from 'next/headers'

export async function delete(data: FormData) {
  ;(await cookies()).delete('name')
}
```

## Cookie behavior

**In Server Components**: Reading cookies works because you access cookie data the browser sends in HTTP request headers. Setting cookies is not supported during rendering.

**In Server Functions**: After setting or deleting a cookie, Next.js returns both updated UI and new data in a single server roundtrip.
