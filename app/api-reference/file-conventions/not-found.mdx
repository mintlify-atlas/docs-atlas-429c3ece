title: not-found.js
description: API reference for the not-found.js file convention, which renders UI when the notFound function is called in a route segment.
---

Next.js provides two conventions for handling not found cases:

- **`not-found.js`**: Renders when the `notFound()` function is called in a route segment.
- **`global-not-found.js`**: Defines a global 404 page for unmatched routes. Handled at the routing level, bypassing normal rendering.

## `not-found.js`

The `not-found` file renders UI when `notFound()` is thrown in a route segment. Next.js returns a `200` HTTP status for streamed responses and `404` for non-streamed responses.

```tsx filename="app/not-found.tsx"
import Link from 'next/link'

export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>Could not find requested resource</p>
      <Link href="/">Return Home</Link>
    </div>
  )
}
```

## `global-not-found.js`

<Note>The `global-not-found.js` file is experimental. Enable it with `experimental.globalNotFound: true` in `next.config.ts`.</Note>

Unlike `not-found.js`, this is used when a requested URL doesn't match any route. It bypasses normal rendering, so you need to import any global styles, fonts, or dependencies your 404 page requires.

Useful when:
- Your app has multiple root layouts (e.g., `app/(admin)/layout.tsx` and `app/(shop)/layout.tsx`).
- Your root layout uses top-level dynamic segments (e.g., `app/[country]/layout.tsx`).

Enable with:

```tsx filename="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    globalNotFound: true,
  },
}

export default nextConfig
```

Create `app/global-not-found.js` with a full HTML document:

```tsx filename="app/global-not-found.tsx"
import './globals.css'
import { Inter } from 'next/font/google'
import type { Metadata } from 'next'

const inter = Inter({ subsets: ['latin'] })

export const metadata: Metadata = {
  title: '404 - Page Not Found',
}

export default function GlobalNotFound() {
  return (
    <html lang="en" className={inter.className}>
      <body>
        <h1>404 - Page Not Found</h1>
        <p>This page does not exist.</p>
      </body>
    </html>
  )
}
```

## Props

`not-found.js` and `global-not-found.js` components do not accept any props.

<Note>The root `app/not-found.js` and `app/global-not-found.js` files handle any unmatched URLs for your whole application.</Note>

## Examples

### Data fetching

`not-found` is a Server Component by default. You can mark it `async` to fetch data:

```tsx filename="app/not-found.tsx"
import Link from 'next/link'
import { headers } from 'next/headers'

export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>Could not find requested resource</p>
      <p>
        View <Link href="/blog">all posts</Link>
      </p>
    </div>
  )
}
```

### Metadata

For `global-not-found.js`, export a `metadata` object or `generateMetadata` function:

```tsx filename="app/global-not-found.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Not Found',
  description: 'The page you are looking for does not exist.',
}

export default function GlobalNotFound() {
  return (
    <html lang="en">
      <body>
        <h1>Not Found</h1>
        <p>The page you are looking for does not exist.</p>
      </body>
    </html>
  )
}
```
