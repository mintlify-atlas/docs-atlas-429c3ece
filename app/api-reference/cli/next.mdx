The Next.js CLI develops, builds, starts your application, and more.

<Tabs>
  <Tab title="pnpm">
    ```bash
    pnpm next [command] [options]
    ```
  </Tab>
  <Tab title="npm">
    ```bash
    npx next [command] [options]
    ```
  </Tab>
  <Tab title="yarn">
    ```bash
    yarn next [command] [options]
    ```
  </Tab>
  <Tab title="bun">
    ```bash
    bunx next [command] [options]
    ```
  </Tab>
</Tabs>

<Note>With `npm run`, use `--` before CLI flags so npm forwards them to `next`. Not required for pnpm, yarn, or bun.</Note>

## Global options

| Option | Description |
| --- | --- |
| `-h` or `--help` | Shows all available options |
| `-v` or `--version` | Outputs the Next.js version number |

## Commands

| Command | Description |
| --- | --- |
| `dev` | Starts Next.js in development mode with HMR, error reporting, and more. |
| `build` | Creates an optimized production build. |
| `start` | Starts Next.js in production mode. Requires `next build` first. |
| `info` | Prints system details for reporting Next.js bugs. |
| `telemetry` | Enable or disable anonymous telemetry collection. |
| `typegen` | Generates TypeScript definitions for routes without a full build. |
| `upgrade` | Upgrades your Next.js application to the latest version. |
| `experimental-analyze` | Analyzes bundle output using Turbopack. |

<Note>Running `next` without a command is an alias for `next dev`.</Note>

## `next dev` options

Starts the application in development mode with Hot Module Reloading (HMR), error reporting, and more.

<Note>Development builds output to `.next/dev` instead of `.next`. This allows `next dev` and `next build` to run concurrently.</Note>

| Option | Description |
| --- | --- |
| `-h, --help` | Show all available options |
| `[directory]` | Directory to build. Defaults to current directory. |
| `--turbopack` | Force enable Turbopack (default). Also `--turbo`. |
| `--webpack` | Use Webpack instead of Turbopack. |
| `-p` or `--port <port>` | Port number. Default: 3000. |
| `-H` or `--hostname <hostname>` | Hostname. Default: 0.0.0.0. |
| `--experimental-https` | Start with HTTPS and a self-signed certificate. |
| `--experimental-https-key <path>` | Path to HTTPS key file. |
| `--experimental-https-cert <path>` | Path to HTTPS certificate file. |
| `--experimental-https-ca <path>` | Path to HTTPS certificate authority file. |
| `--experimental-cpu-prof` | Enable CPU profiling. Saves to `.next/cpu-profiles/`. |

## `next build` options

Creates an optimized production build. The output displays information about each route:

```bash filename="Terminal"
Route (app)
┌ ○ /_not-found
└ ƒ /products/[id]

○  (Static)   prerendered as static content
ƒ  (Dynamic)  server-rendered on demand
```

| Option | Description |
| --- | --- |
| `-h, --help` | Show all available options |
| `[directory]` | Directory to build. Defaults to current directory. |
| `--turbopack` | Force enable Turbopack (default). Also `--turbo`. |
| `--webpack` | Build using Webpack. |
| `-d` or `--debug` | Enables verbose build output including rewrites, redirects, and headers. |
| `--profile` | Enable production profiling for React. |
| `--no-lint` | Disable linting during build. |
| `--no-mangling` | Disable name mangling. Affects performance; for debugging only. |
| `--experimental-app-only` | Build only App Router routes. |
| `--debug-prerender` | Debug prerender errors in development. |
| `--debug-build-paths=<patterns>` | Build only specific routes for debugging. |
| `--experimental-cpu-prof` | Enable CPU profiling. |

## `next start` options

Starts the application in production mode. Requires `next build` first.

| Option | Description |
| --- | --- |
| `-h` or `--help` | Show all available options |
| `[directory]` | Directory to start. Defaults to current directory. |
| `-p` or `--port <port>` | Port number. Default: 3000. |
| `-H` or `--hostname <hostname>` | Hostname. Default: 0.0.0.0. |
| `--keepAliveTimeout <ms>` | Maximum milliseconds to wait before closing inactive connections. |
| `--experimental-cpu-prof` | Enable CPU profiling. |

## `next info` options

Prints relevant system details for reporting Next.js bugs on [GitHub](https://github.com/vercel/next.js/issues).

```bash filename="Terminal"
Operating System:
  Platform: darwin
  Arch: arm64
Binaries:
  Node: 20.12.0
  npm: 10.5.0
Relevant Packages:
  next: 15.0.0
  react: 19.0.0
  typescript: 5.5.4
```

| Option | Description |
| --- | --- |
| `-h` or `--help` | Show all available options |
| `--verbose` | Collect additional debug information. |

## `next typegen` options

Generates TypeScript definitions for routes, pages, layouts, and route handlers without running a full build.

| Option | Description |
| --- | --- |
| `-h` or `--help` | Show all available options |
| `[directory]` | Directory of the Next.js application. |

## `next telemetry` options

Enables or disables completely anonymous telemetry collection.

```bash
next telemetry disable
next telemetry enable
next telemetry status
```

## `next upgrade` options

Upgrades your Next.js application to the latest version.

| Option | Description |
| --- | --- |
| `-h` or `--help` | Show all available options |
| `--canary` | Upgrade to the latest canary version. |
