Welcome to the Next.js App Router documentation! This section will help you build your first Next.js application and learn the core features you'll use in every project.

## What is the App Router?

The App Router is Next.js's modern routing system built on React Server Components. It supports nested layouts, streaming, Server Actions, and the `use cache` directive for fine-grained caching.

## Pre-requisite knowledge

The documentation assumes familiarity with:

- HTML and CSS
- JavaScript (ES6+)
- React fundamentals (components, props, state)

If you're new to React, start with the [React Foundations course](https://nextjs.org/learn/react-foundations) before proceeding.

## Learning path

<Steps>
  <Step title="Install and create a project">
    Use `create-next-app` to scaffold a new Next.js application with TypeScript, Tailwind CSS, and ESLint configured.
  </Step>
  <Step title="Understand project structure">
    Learn the file and folder conventions, including routing files like `layout.tsx`, `page.tsx`, `loading.tsx`, and `error.tsx`.
  </Step>
  <Step title="Build pages and layouts">
    Create your first pages and shared layouts using file-system routing.
  </Step>
  <Step title="Fetch and display data">
    Use async Server Components to fetch data directly from databases or APIs.
  </Step>
  <Step title="Handle mutations">
    Use Server Actions to mutate data and update the UI in a single server roundtrip.
  </Step>
  <Step title="Deploy your application">
    Deploy to Vercel, a Node.js server, or export as a static site.
  </Step>
</Steps>
