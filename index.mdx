## What is Next.js?

Next.js is a React framework for building full-stack web applications. You use React Components to build user interfaces, and Next.js for additional features and optimizations including routing, data fetching, rendering strategies, and more.

Next.js automatically configures lower-level tools like bundlers and compilers, so you can focus on building your product and shipping quickly.

## Key features

<CardGroup cols={3}>
  <Card title="Server Components" icon="server">
    Render components on the server to reduce JavaScript sent to the browser and improve performance.
  </Card>
  <Card title="File-system Routing" icon="folder-tree">
    Create routes by adding files to your `app/` or `pages/` directory — no configuration needed.
  </Card>
  <Card title="Data Fetching" icon="database">
    Fetch data with `async/await` in Server Components or use `getServerSideProps` and `getStaticProps`.
  </Card>
  <Card title="Built-in Optimizations" icon="bolt">
    Automatic image, font, and script optimization out of the box.
  </Card>
  <Card title="Turbopack" icon="zap">
    The fast Rust-based bundler for lightning-fast local development and builds.
  </Card>
  <Card title="TypeScript" icon="brackets-curly">
    First-class TypeScript support with zero configuration required.
  </Card>
</CardGroup>

## Start building

<Steps>
  <Step title="Create a new project">
    Use `create-next-app` to scaffold a new application:

    <CodeGroup>
    ```bash npm
    npx create-next-app@latest my-app
    ```
    ```bash yarn
    yarn create next-app my-app
    ```
    ```bash pnpm
    pnpm create next-app my-app
    ```
    </CodeGroup>
  </Step>
  <Step title="Run the development server">
    ```bash
    cd my-app
    npm run dev
    ```

    Open [http://localhost:3000](http://localhost:3000) to see your application.
  </Step>
  <Step title="Start editing">
    Edit `app/page.tsx` (App Router) or `pages/index.tsx` (Pages Router) to see live updates thanks to Fast Refresh.
  </Step>
</Steps>
