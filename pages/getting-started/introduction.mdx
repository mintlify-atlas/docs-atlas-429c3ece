The **Pages Router** uses an intuitive file-system router to map each file to a route. It is still fully supported in all current versions of Next.js.

<Note>
  You're viewing the documentation for the Pages Router. See the [App Router](/app/getting-started/introduction) documentation for the latest features including React Server Components, streaming, and more.
</Note>

## What is the Pages Router?

Before Next.js 13, the Pages Router was the main way to create routes in Next.js. It works by mapping files in the `pages/` directory to URL routes automatically.

- `pages/index.js` → `/`
- `pages/about.js` → `/about`
- `pages/blog/[slug].js` → `/blog/:slug`

## How navigation works

The Pages Router provides client-side navigation between pages using the `<Link>` component from `next/link`. When a link enters the user's viewport, Next.js prefetches the page in the background, making navigation nearly instant.

## Rendering strategies

The Pages Router supports multiple rendering strategies per page:

| Strategy | Function | When to use |
|---|---|---|
| Static Generation | `getStaticProps` | Data available at build time |
| Static Generation + dynamic paths | `getStaticPaths` | Dynamic routes with static data |
| Server-side Rendering | `getServerSideProps` | Data must be fresh on each request |
| Client-side Rendering | `useEffect` / SWR | User-specific or frequently updated data |
